## Predicates under test

The existing general vector BLAS gate is:

```text
core_work >= 4096
```

Below that gate, the portable small-vector baseline is:

```text
core_work >= 1024
batch >= 4
```

The reuse-aware extension is:

```text
core_work >= 576
reuse_intensity = batch * output_extent
reuse_intensity >= 128
```

Because `core_work = K * output_extent`, the second condition is
equivalent to comparing total contracted work with the `K` values
copied by vector packing.

The extension adds work to the baseline predicate.  It does not
disable an existing pack-once selection.

For the negative or zero-stride vectors in this notebook, the
automatic pack range is the general gate OR the baseline OR the
reuse extension.  That complete range was **implemented** for
this run (`general_blas_or_combined`).
